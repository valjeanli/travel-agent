## 1. 研究員（Researcher）

| 項目 | 內容 |
|------|------|
| **檔案名稱** | `researcher` |
| **職責** | 針對特定旅遊主題進行深度資料搜集與分析 |
| **輸出格式** | 結構化 Markdown 研究報告（含表格、價格、建議） |
| **常用工具** | 網路搜尋、瀏覽器、終端機 |

### 子類別與負責主題

研究員會根據任務需求被分配不同的研究主題：

| 主題 | 輸出檔案 | 內容涵蓋 |
|------|---------|---------|
| 🏙️ **城市路線與景點** | `taiwan-attractions.md` | 城市推薦路線、必去景點、一日遊建議、行程總表雛形、天氣與季節考量 |
| ✈️ **航班研究** | `taiwan-flights.md` | 來回機票價格比較、航空公司選擇、行李政策、訂票建議 |
| 🏨 **住宿研究** | `taiwan-accommodation.md` | 各城市住宿推薦、價格區間、地理位置分析、性價比評比 |
| 🍜 **餐飲研究** | `taiwan-food.md` | 在地美食推薦、夜市攻略、餐廳預算、飲食文化特色 |
| 🚇 **後勤交通研究** | `taiwan-logistics.md` | 城市間交通方式、高鐵/台鐵/客運價格與時間、簽證資訊、網路/eSIM、旅遊保險 |

### 研究員工作原則

1. **資料來源多樣化** — 交叉比對至少 3 個來源（Google Flights、Agoda/Booking.com、官方網站、旅遊部落格）
2. **價格即時性** — 標註資料日期與貨幣單位
3. **結構化輸出** — 使用表格呈現比較資訊，便於後續任務消費
4. **香港旅客視角** — 以持特區護照/BNO 的香港居民為對象
5. **保守估算** — 價格預估取中高值，避免低估實際開支

## 2. 規劃師（Planner）

| 項目 | 內容 |
|------|------|
| **檔案名稱** | `planner` |
| **職責** | 綜合所有研究成果，產出最終分鐘級行程計劃 |
| **輸出格式** | 完整 Markdown 行程書（含每日時程、交通、費用、備註） |
| **依賴** | 所有研究員任務完成後啟動 |

### 規劃師工作原則

1. **分鐘級時程** — 每日行程精確到 30 分鐘區間，包含交通時間緩衝
2. **備案思維** — 每個時段提供雨天/替代方案
3. **資訊密度最大化** — 在同一表格中呈現時間、活動、交通方式、費用、備註
4. **鏈接研究成果** — 在行程中交叉引用住宿/餐飲/交通研究的具體建議
5. **可執行性優先** — 行程需考慮體力分配、行李寄放、Check-in/out 時間等實際因素
6. **成本透明** — 標註每一項目的價格，讓旅客清楚總開支分佈

### 輸出文件結構

```
# 行程標題 + 基本資訊表格
# 主要住宿安排
## Day N — 日期：主題
### 上午時間段
### 中午時間段
### 下午時間段
### 傍晚時間段
### 夜晚時間段
# Day X 總結（當日總開支、省錢小提示）
# 最終建議與提醒
```

## 3. 預算審查員（Budget Reviewer）

| 項目 | 內容 |
|------|------|
| **檔案名稱** | `budget-reviewer` |
| **職責** | 綜合所有研究報告的價格數據，產出總預算報告 |
| **輸出格式** | 結構化 Markdown 預算書（含分類明細、方案對比、可行性檢查） |
| **依賴** | 所有研究員任務完成後啟動 |

### 預算審查員工作原則

1. **三方案制** — 提供經濟型、標準型（推薦）、豪華型三種預算方案
2. **可行性檢查** — 確認總預算是否足夠，並顯示可用餘額
3. **佔比分析** — 計算各類開支佔總預算比例
4. **匯率透明** — 標註使用匯率與資料日期
5. **交叉驗證** — 確保各研究報告之間的價格一致性
6. **出發前預付清單** — 列出需提前付款的項目（機票、住宿、高鐵早鳥票、eSIM、保險）

### 預算書結構

```
# 總預算標題
## 一、可行性檢查
## 二、預算比例基準
## 三、各類開支明細（住宿、餐飲、交通、活動、雜項）
## 四、方案對比
## 五、現金管理建議
## 六、必要開支一覽（出發前必付）
## 七、總結對比 + 省錢秘訣
```

## 4. 發布者（Publisher）

| 項目 | 內容 |
|------|------|
| **檔案名稱** | `publisher` |
| **職責** | 將最終產出發布至 GitHub 儲存庫 |
| **輸出格式** | Git 提交 + 推送 |
| **依賴** | 規劃師和預算審查員任務完成後啟動 |

### 發布者工作原則

1. **保持結構** — 維持 `/trips/`、`/resources/`、`/docs/` 三層目錄結構
2. **檔案命名慣例** — 行程檔：`<城市>-<日期>.md`；資源檔：`<主題>.md`
3. **提交訊息格式** — `trip: <城市> <日期> — <簡短摘要>`
4. **完整性檢查** — 確保所有預期檔案都已就位，沒有缺失
5. **發布前審查** — 確認檔名、路徑、內容品質
6. **語言一致性** — 所有內容使用繁體中文

### GitHub 儲存庫結構

```
travel-agent/

│   └── taiwan-8day-may-2026.md
├── resources/          # 研究資源—住宿、航班、餐飲等
│   ├── taiwan-budget.md
│   ├── taiwan-flights.md
│   ├── taiwan-accommodation.md
│   ├── taiwan-food.md
│   ├── taiwan-attractions.md
│   └── taiwan-logistics.md

    ├── profiles.md
    └── workflow.md
```
