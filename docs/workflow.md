## 階段一：任務拆解

| 步驟 | 負責人 | 動作 |
|------|--------|------|
| 1.1 | **規劃師（Orchestrator）** | 接收高層級目標（「規劃一個 8 天台灣之旅，預算 HK$10,000，從香港出發」） |
| 1.2 | **規劃師** | 拆解為獨立研究主題（航班、住宿、餐飲、景點路線、後勤交通、預算） |
| 1.3 | **規劃師** | 透過 `kanban_create` 建立子任務，指定 `researcher` 檔案 |
| 1.4 | **規劃師** | 設定任務依賴關係，確保後續任務按順序啟動 |

### 任務依賴圖

```
航班研究 ──┐
城市景點 ──┼──→ 住宿研究 ──┐
餐飲研究 ──┤              │
後勤交通 ──┼──→ 預算匯總 ──┼──→ 最終行程 ──→ GitHub發布
            │              │
            └──→ 住宿研究 ──┘
```

- **獨立任務**（可並行）：航班、城市景點、餐飲、後勤交通
- **依賴任務**：住宿依賴城市景點（需知道去哪些城市）
- **聚合任務**：預算匯總依賴所有研究；最終行程依賴預算匯總

## 階段二：平行研究

所有研究員任務可同時進行（除住宿需等待城市景點結果）。

### 研究任務啟動條件

| 任務 | 啟動條件 | 預計產出時間 |
|------|---------|------------|
| 城市景點研究 | 立即啟動 | ~10 分鐘 |
| 航班研究 | 立即啟動 | ~8 分鐘 |
| 後勤交通研究 | 立即啟動 | ~10 分鐘 |
| 餐飲研究 | 立即啟動 | ~8 分鐘 |
| 住宿研究 | 城市景點研究完成 | ~8 分鐘 |
| 預算匯總 | 所有研究完成 | ~5 分鐘 |
| 最終行程組裝 | 預算匯總完成 | ~10 分鐘 |

### 研究員工作流程

1. 接收任務，讀取任務描述與依賴任務的輸出
2. 使用網路搜尋獲取最新價格與資訊
3. 交叉比對多個來源的數據
4. 以香港旅客角度撰寫結構化報告
5. 完成後呼叫 `kanban_complete`，將產出傳遞給下游

## 階段三：匯總與合成

### 預算審查

| 步驟 | 動作 |
|------|------|
| 3.1 | 收集所有研究報告中的價格數據 |
| 3.2 | 統一貨幣單位（HKD 為主，TWD 為輔） |
| 3.3 | 計算各類開支佔比，與建議比例比對 |
| 3.4 | 產出三種預算方案（經濟、標準、豪華） |
| 3.5 | 檢查 HK$10,000 總預算的可行性 |
| 3.6 | 列出出發前必付開銷 |
| 3.7 | 完成後啟動最終行程任務 |

### 最終行程組裝

| 步驟 | 動作 |
|------|------|
| 3.8 | 讀取所有研究成果（含預算報告） |
| 3.9 | 將行程細化至分鐘級時程 |
| 3.10 | 每天分為上午、中午、下午、傍晚、夜晚五個時段 |
| 3.11 | 在每個時段標註：時間、活動、交通方式、費用、備註 |
| 3.12 | 整合住宿推薦到對應日期的行程中 |
| 3.13 | 加入雨天備案與行李寄放策略 |
| 3.14 | 撰寫最終建議（訂票提醒、高鐵早鳥、現金管理等） |

## 技術細節

### 使用的 Hermes Kanban 工具

| 工具 | 用途 |
|------|------|
| `kanban_create` | 建立子任務（Fan-out） |
| `kanban_show` | 讀取任務詳情與上游產出 |
| `kanban_comment` | 留下跨任務備註 |
| `kanban_heartbeat` | 長時間任務的心跳通知 |
| `kanban_complete` | 完成任務，傳遞摘要與 metadata |
| `kanban_block` | 遇到不明確事項時阻擋等待決策 |
| `kanban_link` | 設定任務間依賴關係 |

### 工作區類型

所有任務使用 `scratch` 工作區（臨時目錄），任務完成後自動清理。發布者使用 `scratch` 目錄進行檔案整理，最終產出推送至 GitHub 永久儲存。

### 語言規範

所有與旅行相關的內容使用 **繁體中文（正體中文）**，以符合目標旅客（香港居民）的閱讀習慣。檔案名稱與程式碼相關內容使用英文。
